# brain-dump v0.2.0 → v1.0 Release Handoff

## Project Overview

**brain-dump** is an opinionated restic wrapper for backing up AI agent memory directories (OpenClaw, Claude Code, Codex, custom workspaces) to encrypted cloud storage (B2, S3, or local). It's a single bash script (~1400 lines) with four lib files, five presets, and a test suite.

**Repository:** https://github.com/philipbankier/brain-dump
**Branch:** `feature/v0.2.0` (2 commits ahead of `main`)
**License:** MIT
**Owner:** Philip Bankier (@philipbankier)

## Key Patterns & Conventions

- **`set -euo pipefail`** at top of every file
- Functions prefixed with `bb::namespace::` (e.g. `bb::config::load`, `bb::telemetry::record`)
- Error output via `bb::output::error "msg"` (goes to stderr)
- Success output via `bb::output::print "✅ msg"` (suppressed by --quiet)
- JSON mode: `BB_JSON_MODE="true"` → all output as JSON via jq
- Config parsed with `yq eval`, output formatted with `jq`
- Restic commands always use `--json` flag, output parsed with `jq`
- Presets define paths + exclude patterns, resolved at snapshot time
- Telemetry stored as JSONL at `~/.brain-dump/telemetry.jsonl`
- Errors quarantined as individual JSON files at `~/.brain-dump/errors/`
- Config at `~/.config/brain-dump/config.yaml`
- Dependencies: bash 3.2+, restic 0.16+, yq 4.0+, jq 1.6+

## Important Gotchas

1. **Config path is `~/.config/brain-dump/config.yaml`** (not `brain-backup`). The rename happened in v0.2.0. Tests and any docs referencing `brain-backup` must be updated.

2. **Config version**: v0.2.0 accepts both version 1 and version 2 in validation. New installs generate version 2. The version bump is non-breaking.

3. **Restic output parsing**: restic outputs multiple JSON lines during backup; the last one with `"message_type":"summary"` is the actual summary. Code uses `grep '"message_type":"summary"' | tail -1`.

4. **Prune output parsing**: restic forget returns JSON that can be an object or array depending on version. Code handles both: `jq -r 'if type == "object" then (.remove | length) else 0 end'`. There was a bug with multiline jq output — fixed with `tr -d '\n' | head -1 | grep -oE '^[0-9]+'`.

5. **Stale restic locks**: If a backup is interrupted, restic leaves a lock. `brain-dump doctor` checks repo reachability but doesn't auto-unlock. Manual fix: `restic unlock`.

6. **Launchd doesn't load ~/.zshenv**: The plist uses a wrapper script (`~/bin/brain-dump-launchd-wrapper.sh`) that sources `~/.zshenv` before running brain-dump. This is how `RESTIC_PASSWORD` and B2 credentials get to the scheduled job.

7. **Per-profile lenient mode**: The `error_handling.mode` config field is read but NOT YET implemented in the snapshot command. Currently all profiles are backed up in a single restic call regardless of mode setting. Implementing this is part of the v1.0 scope.
